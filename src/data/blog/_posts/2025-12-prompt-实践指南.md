# Prompt 拆分标准与工具指南

## 1. System Prompt vs User Prompt 标准拆分规则

核心判断逻辑：**“静态 (配置)” vs “动态 (触发)”**

| 维度 | System Prompt (系统层/配置层) | User Prompt (用户层/触发层) |
| :--- | :--- | :--- |
| **本质定义** | **立宪 / 规则**<br>定义 AI 的身份、底线、世界观和输出格式。 | **提案 / 任务**<br>具体的任务请求、输入数据和变量。 |
| **变化频率** | **常量 (Constant)**<br>在应用的生命周期中几乎不变，类似代码配置。 | **变量 (Variable)**<br>每次交互都在变化。 |
| **包含内容** | 1. **身份**: "你是资深律师..."<br>2. **风格**: "语气专业且冷峻..."<br>3. **格式**: "只输出 JSON..."<br>4. **限制**: "严禁编造事实..."<br>5. **工具**: "你有搜索工具..." | 1. **具体指令**: "分析这份合同..."<br>2. **输入数据**: (合同文本内容)<br>3. **具体参数**: "重点关注第3条款..." |
| **最佳实践** | 此处指令具备**最高优先级**和**全剧覆盖权**。 | 此处内容仅作为**被处理的对象**。 |

## 2. Prompt 调试与自动化工具矩阵

| 工具名称 | 核心定位 | 能力与特点 | 适用场景 | 推荐指数 |
| :--- | :--- | :--- | :--- | :--- |
| **1. OpenAI Playground**<br>*(或 Anthropic Console)* | **原生沙盒** | **【手动调试】**<br>提供直观的左右分栏界面（System/User），最适合快速验证 Prompt 逻辑。 | **入门首选**<br>适合刚开始理解拆分概念，或快速测试简单的 Prompt。 | ⭐⭐⭐⭐⭐<br>(必试) |
| **2. LangSmith**<br>*(by LangChain)* | **调试监控** | **【可视化回放】**<br>记录每一次 LLM 调用，清晰展示 System 指令是否被 User 输入“污染”或覆盖。 | **工程化开发**<br>开发实际应用时，用于排查 Bug 和管理 Prompt 版本。 | ⭐⭐⭐⭐<br>(必备) |
| **3. DSPy**<br>*(Stanford)* | **自动编译** | **【全自动生成】**<br>你只需定义输入/输出和逻辑，它通过算法**自动编写和拆分**最优 Prompt。 | **进阶/自动化**<br>不想手动写 Prompt，希望用编程思维解决复杂问题。 | ⭐⭐⭐<br>(未来趋势) |
| **4. Microsoft Prompt Flow** | **流程编排** | **【结构化设计】**<br>强制使用模板语法 (Jinja2)，将 User Input 变为变量，强迫开发者进行分离。 | **企业级应用**<br>需要批量测试、评估 Prompt 效果的团队。 | ⭐⭐⭐ |
| **5. Fabric**<br>*(Open Source)* | **模式库** | **【最佳范本】**<br>开源的 System Prompt 仓库，提供大量高质量的“人设”模板。 | **个人效能/参考**<br>想学习“顶级的 System Prompt 是怎么写的”。 | ⭐⭐⭐⭐ |

## 3. 推荐路线图

1.  **STEP 1: 立即尝试 [OpenAI Playground]**
    不要安装任何东西，直接去网页版。把你的 Prompt 粘贴进去，手动把“背景信息”剪切到 System 框，把“具体问题”留在 User 框。这是建立直觉最快的方法。

2.  **STEP 2: 参考学习 [Fabric]**
    去 GitHub 看看 Fabric 项目的 `patterns` 文件夹。你会惊讶地发现，他们的 System Prompt 写得多么详细和结构化，这会给你很好的灵感。

3.  **STEP 3: 落地开发 [LangSmith]**
    当你开始写代码调用 API 时，接入 LangSmith。这样你就能看到真实用户输入是如何与你的 System Prompt 交互的。
